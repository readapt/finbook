## 一、产品定位

### 1.1 目标用户

| 用户类型 | 规模 | 特征 | 需求 |
|----------|------|------|------|
| **微型企业** | <10 人 | 无专职会计，成本敏感 | 简单记账、合规报税 |
| **小型企业** | 10-50 人 | 有兼职会计 | 完整财务、合规压力大 |
| **个体户/工作室** | 1-5 人 | 自由职业者 | 收支记录、发票管理 |

**排除用户**: 代账公司（需商业许可）

### 1.2 核心价值

| 价值 | 说明 |
|------|------|
| **数据自主** | 用户自托管，财务数据完全掌控 |
| **零订阅费** | 开源免费，无持续费用 |
| **便捷入口** | Telegram/企业微信/钉钉，无需安装新 App |
| **合规保证** | 符合财政部 2025 新规范 |

### 1.3 部署方式

- **验证期**: 用户自托管（Telegram 入口）
- **正式期**: 用户自托管（企业微信 + 钉钉 + 微信小程序）

### 4.2 数据一致性

| 维度 | 策略 |
|------|------|
| **数据源** | 文本文件（.beancount）为唯一权威数据源 |
| **数据库** | SQLite（默认）+ PostgreSQL（可选），作为缓存 |
| **一致性** | 最终一致性 + 文件锁 + 定期校验 |
| **并发控制** | 文件锁机制 |
| **恢复机制** | 从文本文件重建数据库 |

### 4.3 技术选型

| 组件 | 选型 | 说明 |
|------|------|------|
| **记账引擎** | Beancount v3 | 双式记账核心 |
| **开发语言** | Python 3.10+ | 与 Beancount 一致 |
| **数据库** | SQLite + PostgreSQL 可选 | 灵活部署 |
| **入口集成** | OpenClaw 多通道 | Telegram/企业微信/钉钉 |
| **OCR 服务** | 用户自选（阿里/腾讯/百度/自建） | 灵活性 |

## 五、合规要求

### 5.1 财政部规范（2025 新版）

| 要求 | 实现方案 |
|------|----------|
| **不可逆记账** | 凭证只能红字冲销，不能删除/修改 |
| **电子凭证** | 支持数电票、OFD、XML、PDF 验真 |
| **数据导出** | 标准 CSV/Excel，完整账套 |
| **操作留痕** | 完整审计日志（用户/时间/IP/操作） |
| **境内备份** | 服务器必须在境内，或境内有副本 |

### 5.2 金税四期对接

| 要求 | 实现方案 |
|------|----------|
| **发票验真** | 税务总局全国增值税发票查验平台 API |
| **数电票** | PDF+XML 双文件保存 |
| **一键报税** | 各省市电子税务局 API |
| **三流一致** | 发票流/资金流/业务流匹配 |
